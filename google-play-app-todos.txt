Tradesman — Google Play (Android) release checklist
====================================================

CODEBASE (this repo) — locked in for first Play upload
- Run `npm run mobile:sync` after each web release so `dist` copies into `android/` (set production `VITE_*` in `.env` / CI before this so the bundle talks to prod Supabase, Helcim, etc.).
- Open Android Studio: `npm run mobile:open:android` (or `npx cap open android`).
- Package / app id: `com.tradesman.app` (`applicationId`, `capacitor.config.ts` appId, Play Console must match).
- WebView content: bundled `dist` only (no `server.url`); cleartext HTTP disabled on `<application android:usesCleartextTraffic="false">`.
- `android/app/build.gradle`: `versionName` is read from repo root `package.json`; bump `versionCode` for every Play upload.
- Release type: `debuggable false`, signing via `android/keystore.properties` + `.jks` (see `keystore.properties.example`); `npm run mobile:build:android` produces `app-release.aab` when the keystore exists.
- Icons / splash: adaptive launcher + splash theme in `android/app/src/main/res` (regenerate with `npm run icons:generate` after changing `icon.png`).
- Manifest permissions: INTERNET, location (maps), POST_NOTIFICATIONS, FCM-related — align Play “Data safety” with what you actually collect (Supabase profile, push token, optional GPS per user opt-in).
- Minify: currently off (`minifyEnabled false`); enable R8 only after testing — not required for Play.
- targetSdk / compileSdk: see `android/variables.gradle` (keep meeting Play’s floor when Console warns).

GOOGLE PLAY CONSOLE
- Create app; package name must match `applicationId` / `appId`.
- Complete store listing: title, short/full description, screenshots (phone + 7" tablet if required), feature graphic, privacy policy URL.
- Content rating questionnaire; target audience; ads declaration; data safety form (match what Supabase/Twilio/your APIs collect).
- App access: if login required, provide test credentials or demo mode instructions.
- Signing: use Play App Signing; upload first release as AAB (Android App Bundle).
- Internal testing track first, then closed/open production.

COMPLIANCE & OPS
- Privacy policy hosted URL (web) covering mobile data use, third parties, retention.
- If you use SMS/voice: carrier and regional rules are separate from Play; still disclose in listing and policy.
- Crash reporting (optional): Firebase Crashlytics or similar — declare in data safety.

QUICK COMMANDS
- `npm run mobile:sync` — build web + copy to Android (and iOS if present).
- `npm run mobile:open:android` — open project in Android Studio for Run / Generate Signed Bundle.
